## 项目概述

User Master 是一个用户认证与管理系统，支持多种登录方式（微信公众号登录、GitLab OAuth登录），采用微服务架构设计，包含用户服务、定时任务服务、消息接收服务。

## 技术栈

### 后端
- **语言**: Go 1.20
- **Web框架**: Gin
- **数据库**: MySQL
- **缓存**: Redis
- **对象存储**: 腾讯云COS
- **认证**: JWT
- **RPC**: gRPC

### 前端
- **框架**: Vue 3
- **UI组件库**: Element Plus
- **构建工具**: Vite
- **语言**: TypeScript

## 项目架构

```
user-master/
```


#### 关键代码流程

**1. 获取登录方式 (GetLoginMethods)**

```go
func (c *LoginController) GetLoginMethods(ctx *gin.Context) {
    sys := ctx.DefaultQuery("sys", string(constants.SYS_MEDIAHUB))
    // ...
    // 获取微信二维码
    qrcode, err := c.getWxMpQrCode()
    // ...
    ctx.JSON(http.StatusOK, gin.H{
        "gitlab":    gitlabAuthUrl,
        "wx_qrcode": qrcode,
    })
}
```

**2. 生成微信二维码 (getWxMpQrCode)**

- 1. 获取微信AccessToken (通过gRPC调用crontab服务)
- 2. 调用微信API创建临时二维码 (有效期180秒)
- 3. scene_str = "qr_login"
- 4. 返回二维码ticket和URL

**3. 接收微信消息 (WxReceiveMessage)**

- 1. 验证微信签名
- 2. 解析XML消息
- 3. 处理事件

**4. 处理扫码事件 (wxEventHandler)**

```go
func (c *LoginController) wxEventHandler(msg *Message) error {
    switch strings.ToLower(msg.Event.Value) {
    case "scan":
        // event_key = "qr_login" 或 "qrscene_qr_login"
        if msg.EventKey.Value == string(constants.QRLOGIN) || 
           msg.EventKey.Value == fmt.Sprintf("qrscene_%s", constants.QRLOGIN) {
            // 将 openid 存入 Redis，key 为 ticket，value 为 openid
            key := redis.GetKey(msg.Ticket.Value)
            err := c.redisClient.SetEx(context.Background(), 
                key, msg.FromUserName.Value, time.Minute*3).Err()
        }
    }
    return nil
}
```

**5. 回调处理 (WxCallBackByTicket)**

- 1. 从Redis获取openid
- 2. 查询/创建公众号用户
- 3. 查询/创建主用户，生成头像
- 4. 生成JWT Token
- 5. 设置Cookie，跳转到业务系统
- 6. 返回业务系统跳转地址

```
    │  11. 跳转到业务系统                      │                              │
    └─────────────────────────────────────────────────────────────────────>
```

#### 关键代码流程

**OAuth回调处理 (OAuthCallback)**

- 1. 获取授权码
- 2. 用code换取access_token
- 3. 获取GitLab用户信息
- 4. 查询/创建GitLab用户
- 5. 查询主用户，生成JWT
- 6. 设置Cookie，跳转

### 三、核心组件说明

| 组件 | 说明 |
|------|------|
| `LoginController` | 登录控制器，处理所有登录相关请求 |
| `getWxMpQrCode()` | 调用微信API生成临时二维码 |
| `getWxAccessToken()` | 通过gRPC从crontab服务获取微信AccessToken |
| `WxReceiveMessage()` | 接收并处理微信推送的消息/事件 |
| `wxEventHandler()` | 处理扫码事件，将openid存入Redis |
| `WxCallBackByTicket()` | 扫码后的回调处理，生成JWT |
| `OAuthCallback()` | GitLab OAuth回调处理 |
| `zjwt` | JWT token生成和验证 |

### 四、数据流向

```
用户扫码 → 微信服务器 → 推送扫码事件 → 服务器验证 → 存入Redis(openid)
                                                    ↓
用户确认登录 → 前端轮询/重定向 → 回调接口 → Redis获取openid
                                                    ↓
                                         查询/创建用户 → 生成JWT
                                                    ↓
                                         设置Cookie → 跳转业务系统
```

## 安全注意事项

1. **敏感信息**: 生产环境请修改配置文件中的密码和密钥
2. **JWT密钥**: 生产环境请使用复杂的hashKey
3. **数据库**: 建议使用强密码并限制访问IP
4. **Redis**: 生产环境请启用密码认证
